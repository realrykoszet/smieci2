# ═══════════════════════════════════════════
# 1. BATNA — BEST ALTERNATIVE TO NEGOTIATED AGREEMENT
# ═══════════════════════════════════════════
## Zrodlo: Harvard Negotiation Project (Fisher & Ury, "Getting to Yes")

### DEFINICJA
BATNA = najlepsza opcja ktora masz JESLI negocjacje zawioda. To Twoja sila negocjacyjna.
Im lepsza Twoja BATNA, tym mniejsza presja na zawarcie kiepskiego deal'u.
Im gorsza BATNA klienta, tym wieksza Twoja przewaga.

### NASZA BATNA (Eureka Intelligence)
```
Jesli klient X nie kupi:
→ Mamy 159 innych leadow w pipeline (91 emaili Fala 1 + 30 Fala 2 + 10 Fala 3 + 18 Reports)
→ Mamy 7 produktow gotowych do sprzedazy
→ Mamy 8 regionow geograficznych (PL, DACH, Nordics, Benelux, UK, CEE, Iberia, Global)
→ Mamy darmowe narzedzia ktore buduja autorytet (newsletter, LinkedIn, lead magnet)
→ WNIOSEK: NIGDY nie jestesmy desperacki. Jeden klient to 0.6% pipeline'u.
```

### BATNA KLIENTA (typowy lead)
```
Jesli klient NIE kupi od nas:
→ Opcja A: Big4 (Deloitte/PwC/KPMG/EY) = EUR 50,000-200,000/rok
→ Opcja B: CUBE Global = EUR 500,000/rok (enterprise)
→ Opcja C: Nic nie robic = ryzyko kary EUR 10,000,000 (NIS2/DORA/AI Act)
→ Opcja D: Zrobic wewnetrznie = 6-12 miesiecy, 2-5 FTE, EUR 200,000+
→ WNIOSEK: Kazda opcja klienta jest GORSZA niz nasza oferta (EUR 2,900).
```

### JAK OBLICZYC BATNA KLIENTA (5 krokow)
```
1. Zidentyfikuj regulacje ktore go dotycza (NIS2? DORA? AI Act? CSRD?)
2. Sprawdz deadline enforcement (NIS2 = 03.04.2026, AI Act = 02.08.2026)
3. Oblicz koszt non-compliance (kary + reputacja + utrata klientow)
4. Sprawdz co ma teraz (nic? stary system? Big4?)
5. Porownaj: ich BATNA vs nasza cena → im gorsza ich BATNA, tym silniejsza nasza pozycja
```

### ZASTOSOWANIE W EMAILACH
```
SZABLON — BATNA w pierwszym emailu:
"Wiemy ze [REGULACJA] wchodzi w zycie [DATA]. Typowy koszt compliance przez Big4
to EUR 50,000-200,000. Nasz Discovery Scan daje ten sam insight za EUR 2,900 w 5 dni.
Ryzyko kary za non-compliance: do EUR 10,000,000."

SZABLON — BATNA w follow-up:
"Rozumiemy ze teraz moze nie byc priorytet. Ale Pana/Pani alternatywy to:
Big4 (EUR 50K+), budowa wewnetrzna (6-12 msc) lub brak dzialania (ryzyko EUR 10M).
Nasz pilot to EUR 2,900 i 5 dni — zero ryzyka."
```

### KLUCZOWE ZASADY BATNA
```
1. ZAWSZE znaj swoja BATNA PRZED negocjacja
2. NIGDY nie ujawniaj slabej BATNA ("mamy tylko 3 klientow")
3. POPRAWIAJ swoja BATNA ciagle (wiecej leadow = lepsza BATNA)
4. OBNIAZAJ BATNA klienta (pokaz mu jak jego alternatywy sa gorsze)
5. ZOPA (Zone of Possible Agreement) = przestrzen miedzy Twoja BATNA a ich BATNA
```

# ═══════════════════════════════════════════
# 2. ANALIZA INTERESOW, NIE STANOWISK
# ═══════════════════════════════════════════
## Zrodlo: Harvard Negotiation Project (Fisher & Ury), Interest-Based Bargaining

### DEFINICJA
Stanowisko = CO klient mowi ("Za drogo", "Nie potrzebujemy")
Interes = DLACZEGO to mowi (brak budzetu, strach przed decyzja, brak zrozumienia)
79% firm B2B raportuje krotsze cykle negocjacyjne dzieki interest-based approach.

### DEKODER STANOWISK → INTERESOW (10 najczestszych)

```
STANOWISKO                    | PRAWDZIWY INTERES
─────────────────────────────────────────────────────────────────
"Za drogo"                    | Nie mam budzetu w tym kwartale / Nie widze ROI
"Nie potrzebujemy"            | Nie rozumiem co to robi / Mam inne priorytety
"Muszę zapytac szefa"         | Boje sie podjac decyzje sam / Nie mam authority
"Mamy juz dostawce"           | Nie chce ryzykowac zmiany / Dostawca jest OK
"Nie mamy czasu"              | Zespol jest przeciazony / Wdrozenie wyglada trudne
"Wyslij oferte"               | Chce sie pozbyc sprzedawcy / Potrzebuje doc do szefa
"Odezwe sie za miesiac"       | Nie jest priorytet / Nie wiem jak to uzasadnic
"To nie jest temat teraz"     | Mam pilniejsze problemy / Budzet zamkniety
"Muszę zobaczyc demo"         | Nie ufam opisom / Potrzebuje dowodu
"Ile to kosztuje?"            | Cena jest jedynym kryterium / Nie znam wartosci
```

### 5 PYTAN ODKRYWAJACYCH PRAWDZIWY INTERES

```
1. "Co jest dla Pana/Pani NAJWAZNIEJSZE w tym temacie?"
   → Odkrywa priorytet (cena? czas? bezpieczenstwo? compliance?)

2. "Gdyby cena nie byla problemem — co by Pan/Pani chcial/a osiagnac?"
   → Oddziela stanowisko od interesu

3. "Co musialoby sie wydarzyc zeby to bylo warte Panstwa czasu?"
   → Odkrywa warunki brzegowe klienta

4. "Kto jeszcze w organizacji jest zainteresowany tym tematem?"
   → Identyfikuje decision makera i champion'a

5. "Jakie konsekwencje widzi Pan/Pani jesli nic sie nie zmieni?"
   → Buduje urgency przez interes (strach przed kara/ryzykiem)
```

### ZASTOSOWANIE W SZABLONACH ODPOWIEDZI

```
Klient mowi "Za drogo" → odpowiedz Interest-Based:
"Rozumiem ze budzet jest wazny. Powiedzialby mi Pan wiecej o tym jak
wyglada Panstwa proces compliance teraz? Chce zrozumiec gdzie mozemy
dac NAJWIEKSZA wartosc w ramach Panstwa mozliwosci budzetowych."

Klient mowi "Muszę zapytac szefa" → odpowiedz Interest-Based:
"Jak najbardziej. Czy moge przygotowac krotki one-pager z ROI i
kosztami non-compliance? Cos co ulatwi Panu/Pani prezentacje przed zarzadem."
```

# 3. ETYKIETOWANIE EMOCJONALNE (Tactical Empathy)
# ═══════════════════════════════════════════
## Zrodlo: Chris Voss, "Never Split the Difference" (FBI Hostage Negotiation)

### DEFINICJA
Tactical Empathy = celowe rozpoznanie i nazwanie emocji drugiej strony.
Cel: zbudowac zaufanie, rozbroic opor, uzyskac "That's right".
NIE chodzi o zgadzanie sie — chodzi o ROZUMIENIE.

### MECHANIZM: LABEL → VALIDATE → THAT'S RIGHT
```
1. LABEL: "Wyglada na to ze..." / "Brzmi jakby..." / "Mam wrazenie ze..."
2. VALIDATE: cisza. Czekaj. Nie tluacz sie.
3. THAT'S RIGHT: klient mowi "Dokladnie" / "Tak, wlasnie" / "That's right"
→ To znaczy ze CZUJE SIE ZROZUMIANY. Teraz jest otwarty na Twoja propozycje.
```

### "THAT'S RIGHT" — 2 NAJWAZNIEJSZE SLOWA W NEGOCJACJI (Chris Voss)
```
"That's right" ≠ "You're right"
- "That's right" = "Rozumiesz mnie" → OTWIERA drzwi do deal'u
- "You're right" = "Daj mi spokoj" → ZAMYKA drzwi

Jak uzyskac "That's right":
1. Etykieta + Mirror = Podsumowanie
2. Podsumuj co klient powiedzial JEGO slowami
3. Dodaj etykiete emocjonalna
4. Cisza.
→ Klient: "Dokladnie tak." → BINGO.
```

### 10 GOTOWYCH ETYKIET DLA NASZYCH SCENARIUSZY

```
SCENARIUSZ                              | ETYKIETA
────────────────────────────────────────────────────────────────────────
Klient boi sie NIS2                     | "Wyglada na to ze NIS2 to dodatkowy stres na Panstwa zespol..."
Klient nie ma budzetu                   | "Brzmi jakby budzet byl teraz szczegolnie napiety..."
Klient przeciazony praca                | "Mam wrazenie ze Panstwa zespol juz jest na granicy przepustowosci..."
Klient ma zlego dostawce                | "Wyglada na to ze obecne rozwiazanie nie daje Panstwu spokoju ducha..."
Klient boi sie decyzji                  | "Brzmi jakby ta decyzja wiazala sie z duzym ryzykiem..."
Klient pod presja regulatora            | "Wyglada na to ze regulator daje Panstwu coraz mniej czasu..."
Klient nie ufa nowym dostawcom          | "Mam wrazenie ze zaufanie do nowego partnera to kluczowa kwestia..."
Klient porownuje z Big4                 | "Brzmi jakby mieli Panstwo juz doswiadczenia z duzymi firmami..."
Klient chce demo zanim kupi             | "Wyglada na to ze potrzebujecie zobaczyc wartosc zanim podejmiecie decyzje..."
Klient mowi "nie teraz"                 | "Mam wrazenie ze timing jest teraz wyzwaniem..."
```

### ZASTOSOWANIE W FOLLOW-UP EMAILACH

```
Follow-up #1 (dzien 5):
"[Imie], wyglada na to ze [REGULACJA] to dodatkowe obciazenie na Panstwa zespol.
Wiele firm w Panstwa branzy mierzy sie z tym samym. Dlatego zbudowalismy narzedzie
ktore daje odpowiedzi w 5 dni zamiast 4 tygodni."

Odpowiedz na "Za drogo":
"Brzmi jakby budzet byl teraz szczegolnie napiety. Zupelnie to rozumiemy.
Dlatego nasz Discovery Scan kosztuje EUR 2,900 — nie EUR 50,000 jak Big4.
Czy moge pokazac jak to wyglada na przykladzie firmy podobnej do Panstwa?"
```

### MIRROR (Chris Voss) — uzywaj RAZEM z etykietami
```
Klient: "Nasz compliance jest w porzadku, ale mamy problemy z dokumentacja."
Mirror: "...problemy z dokumentacja?"
→ Klient rozwija temat. Ty sluchasz. Potem dajesz etykiete.
```

# 4. ZARZADZANIE USTEPSTWAMI (Concession Strategy)
# ═══════════════════════════════════════════
## Zrodlo: RED BEAR Negotiation, Simon-Kucher, The Sales Blog

### ZLOTA ZASADA: NIGDY NIE DAWAJ RABATU ZA DARMO
```
Kazde ustepstwo = WYMIANA. Zawsze.
"Moge..." → "...jesli Pan/Pani..."
Klient dostaje cos → Ty dostajesz cos.
Klient nie daje nic w zamian → Ty nie dajesz nic.
```

### STRATEGIA "ZMNIEJSZAJACYCH SIE USTEPSTW"
```
Runda 1: -10% (od EUR 2,900 do EUR 2,610) → ale TYLKO jesli kontrakt 12 msc
Runda 2: -5% (do EUR 2,480)              → ale TYLKO jesli platnosc z gory
Runda 3: -2% (do EUR 2,430)              → i to OSTATNIA mozliwosc
Runda 4: STOP. "To nasza najlepsza oferta."

EFEKT: Klient widzi ze ustepstwa sie zmniejszaja → rozumie ze zbliamy sie do granicy.
```

### NASZA TABELA USTEPSTW

```
CO MOZEMY DAC                     | KIEDY                          | W ZAMIAN ZA CO
──────────────────────────────────────────────────────────────────────────────────
-10% ceny                         | Kontrakt roczny                | 12 msc commitment
-5% ceny                          | Platnosc z gory (100%)         | Cash flow natychmiast
+50 dodatkowych docs w scanie     | Standard plan                  | Case study / testimonial
Raport w 2 jezykach               | Professional plan              | Rekomendacja do 3 firm
Follow-up scan za darmo (1x)      | Pilot purchase                 | Zgoda na uzycie logo
Prioritetowy turnaround (3 dni)   | Enterprise deal                | Kontrakt 24 msc
Dedykowany account manager        | 5+ licencji                    | Volume commitment
1 darmowy Quick Report            | Nowy klient                    | Spotkanie demo z 2 kolegami

CO NIGDY NIE DAJEMY:
✗ Rabat > 20% (niszczenie marzy)
✗ Darmowy pelny scan (dewaluacja produktu)
✗ Rabat bez warunku (desperacja)
✗ Rabat na proste "za drogo" (bez odkrycia interesu)
✗ Obnizenie ceny ponizej EUR 2,000 (Discovery) / EUR 299 (licencja min)
```

### WARUNKOWE USTEPSTWA — GOTOWE ZDANIA

```
"Moge obnizyc cene o 10% jesli podpiszecie na 12 miesiecy zamiast jednorazowo."

"Zamiast rabatu cenowego moge dodac: 50 dodatkowych dokumentow w skanowaniu,
raport w drugim jezyku i follow-up scan za 3 miesiace — gratis."

"Jesli Panstwo zdecyduja sie do piatku, moge zaproponowac prioritetowy turnaround
— wyniki w 3 dni robocze zamiast 5."

"Przy platnosci z gory za caly rok moge dac 5% znizki — to EUR 145 oszczednosci."
```

# 5. BENCHMARKI I KOTWICZENIE (Anchoring)
# ═══════════════════════════════════════════
## Zrodlo: Kahneman & Tversky (Anchoring Bias), Simon-Kucher, Impact Analytics

### DEFINICJA
Anchoring = ludzie opieraja decyzje na PIERWSZEJ informacji ktora dostaja.
Kto PIERWSZY poda cene — ten kontroluje negocjacje.
W B2B: ZAWSZE podawaj najdrozsza opcje PIERWSZA.

### MECHANIZM KOTWICZENIA (4 poziomy)

```
POZIOM 1 — Kotwica na konkurencji:
"Big4 bierze EUR 50,000-200,000 za compliance audit.
CUBE Global kosztuje EUR 500,000/rok.
Nasz Discovery Scan: EUR 2,900."
→ EUR 2,900 WYDAJE SIE tanim w porownaniu z EUR 50K-500K.

POZIOM 2 — Kotwica na ryzyku:
"Kara za non-compliance NIS2: do EUR 10,000,000.
Nasz scan: EUR 2,900. To 0.029% potencjalnej kary."
→ EUR 2,900 = POLISA UBEZPIECZENIOWA za grosze.

POZIOM 3 — Kotwica na opcjach (Good-Better-Best):
"Enterprise Audit: EUR 14,999/rok (unlimited, 24/7, dedicated team)
Professional Scan: EUR 4,999/rok (monthly, 7 regulacji, 2 jezyki)
Discovery Scan: EUR 2,900 (jednorazowy, 125 testow, 1 raport)"
→ Wiekszosc wybierze Professional (srodek). Ale nawet Discovery wyglada jak okazja.

POZIOM 4 — Kotwica na czasie:
"Budowa wewnetrzna: 6-12 miesiecy, 2-5 FTE.
Big4: 4-8 tygodni.
My: 5 dni roboczych."
→ Czas to pieniadz. 5 dni vs 12 miesiecy = JASNE.
```

### ZASTOSOWANIE W KAZDYM EMAILU/OFERCIE

```
SZABLON — Kotwica cenowa NA POCZATKU:
"Firmy w Panstwa branzy wydaja EUR 50,000-200,000 rocznie na compliance audity.
Nasz AI-powered Discovery Scan daje porownywalne wyniki za EUR 2,900 w 5 dni."

SZABLON — Kotwica w ofercie (3 opcje):
"Mamy 3 opcje dopasowane do Panstwa potrzeb:
→ Enterprise (EUR 14,999/rok): unlimited scany, 24/7, dedicated team
→ Professional (EUR 4,999/rok): monthly scany, 7 regulacji, priority support
→ Discovery (EUR 2,900): jednorazowy scan, 125 testow, raport w 5 dni
Ktora opcja najlepiej pasuje do Panstwa obecnych potrzeb?"
```

### ZASADY KOTWICZENIA
```
1. ZAWSZE bądz PIERWSZY z cena/wartoscia
2. NIGDY nie rozpoczynaj od najtanszej opcji
3. Kotwicz na WARTOSCI konkurencji, nie na swojej cenie
4. Uzywaj KONKRETNYCH liczb (EUR 2,900 nie "okolo 3K")
5. Powtarzaj kotw ice w kazdym touchpoincie (email, call, oferta, follow-up)
```

# 6. ASERTYWNOSC I PRAWO DO "NIE"
# ═══════════════════════════════════════════
## Zrodlo: Chris Voss ("Start with No"), RED BEAR, ASLAN Training

### DEFINICJA
Asertywnosc w sprzedazy = prawo do powiedzenia NIE — i klientowi, i sobie.
Desperacja ZABIJA deal'e. Szacunek ZAMYKA deal'e.
"Willing to walk away" = najsilniejsza pozycja negocjacyjna.

### KLIENT MA PRAWO POWIEDZIEC "NIE" — I TO JEST OK
```
Chris Voss: "No" to nie koniec negocjacji — to POCZATEK.
"No" oznacza:
→ "Nie jestem jeszcze gotowy"
→ "Potrzebuje wiecej informacji"
→ "Nie rozumiem wartosci"
→ "Boje sie podjac decyzje"
→ RZADKO: "Absolutnie nigdy"

Po uslyszeniu "No":
1. Etykieta: "Brzmi jakby cos Pana/Pania powstrzymywalo..."
2. Calibrated question: "Co musialoby sie zmienic zeby to mialo sens?"
3. NIE naciskaj. NIE argumentuj. SLUCHAJ.
```

### TY TEZ MASZ PRAWO POWIEDZIEC "NIE"
```
Kiedy mowisz NIE:
→ "Nie obniamy ceny ponizej EUR 2,000." (granica)
→ "Nie robimy darmowych pelnych scanow." (wartosc)
→ "Nie mozemy zagwarantowac wynikow bez danych wejsciowych." (rzetelnosc)
→ "Nie pracujemy na weekendach za standard pricing." (granice)

EFEKT: Klient SZANUJE Cie wiecej. Desperate seller = zero szacunku.
```

### BREAKUP EMAIL — OSTATNI FOLLOW-UP
```
KIEDY: Follow-up #5 (dzien 40) — klient nie odpowiedzial na 4 wiadomosci

SZABLON:
"[Imie],

Pisalem kilka razy w sprawie [TEMAT] ale nie otrzymalem odpowiedzi.
Calkowicie rozumiem ze to moze nie byc temat teraz.

Nie chce byc natretny, wiec to moja ostatnia wiadomosc w tej sprawie.
Jesli kiedykolwiek temat compliance wroci na Panstwa agende — prosze
o kontakt. Zostawiam swoje dane.

Pozdrawiam,
Lukasz Trzeciak
Eureka Intelligence"

DLACZEGO DZIALA:
→ Zero presji = szacunek
→ "Ostatnia wiadomosc" = scarcity (moze zmienic zdanie)
→ "Zostawiam dane" = drzwi otwarte
→ Statystycznie 5-15% breakup emaili generuje odpowiedz
```

# ═══════════════════════════════════════════
# 7. NLP: RAPPORT BUILDING
# ═══════════════════════════════════════════
## Zrodlo: NLP (Bandler & Grinder), Chris Voss (Mirroring), Carew Sales Training

### DEFINICJA
Rapport = podswiadome poczucie "ten czlowiek jest jak ja, moge mu zaufac".
Buduje sie przez: Mirroring + Pacing + Leading + Presupozycje.

### MIRRORING (powtarzanie ostatnich 3 slow)
```
Klient: "Mamy problemy z terminami compliance."
Mirror: "...z terminami compliance?"
→ Klient rozwija temat. Ty SLUCHASZ.

Klient: "To jest dla nas bardzo kosztowne."
Mirror: "...bardzo kosztowne?"
→ Klient wyjasnia CO jest kosztowne. Ty masz intel.
```

# 10. NLP: META MODEL (Precyzyjne Pytania)
# ═══════════════════════════════════════════
## Zrodlo: NLP Meta Model (Bandler & Grinder), 3 kategorie: Deletion, Distortion, Generalization

### DEFINICJA
Meta Model = zestaw precyzyjnych pytan ktore rozbijaja niejasne stwierdzenia.
Ludzie KASUJA informacje (Deletion), ZNIEKSZTALCAJA je (Distortion),
GENERALIZUJA (Generalization). Meta Model je odzyskuje.

### 3 KATEGORIE + PYTANIA SPRZEDAZOWE

```
KATEGORIA 1 — DELETION (brakujaca informacja)
────────────────────────────────────────────────────────
Klient: "To jest za drogie."
Pytanie: "Za drogie w porownaniu do CZEGO?" (specification)
→ Odkrywasz punkt odniesienia klienta.

Klient: "Mamy problemy."
Pytanie: "Jakie DOKLADNIE problemy?" (specification)
→ Odkrywasz konkrety zamiast ogolnikow.

Klient: "To nie dziala."
Pytanie: "Co KONKRETNIE nie dziala? I jak to sprawdziliscie?" (specification)
→ Odkrywasz czy mowia z doswiadczenia czy z przekonania.


KATEGORIA 2 — DISTORTION (znieksztalcenie)
────────────────────────────────────────────────────────
Klient: "Jesli kupimy, to i tak nic sie nie zmieni."
Pytanie: "Co sprawia ze Pan/Pani tak uwaza?" (cause-effect challenge)
→ Rozbijasz zalozenie przyczynowo-skutkowe.

Klient: "Szef nie bedzie zadowolony."
Pytanie: "Skad Pan/Pani wie? Czy rozmawial/a Pan/Pani z nim o tym?" (mind reading challenge)
→ Odkrywasz czy to FAKT czy PROJEKCJA.

Klient: "To oznacza ze nas stac."
Pytanie: "Jak dokladnie jedno prowadzi do drugiego?" (complex equivalence)
→ Rozbijasz uproszczony lacznik logiczny.


KATEGORIA 3 — GENERALIZATION (nadmierne uogolnienie)
────────────────────────────────────────────────────────
Klient: "AI NIGDY nie dziala dobrze."
Pytanie: "Nigdy? Czy zdarzylo sie ze AI dal dobry wynik?" (universal quantifier challenge)
→ Jeden kontrprzyklad rozbija generalizacje.

Klient: "Nie MOZEMY tego kupic."
Pytanie: "Co by sie stalo gdybyscie MOGLI?" (modal operator challenge)
→ Zmienia "nie mozemy" na "co by bylo gdyby".

Klient: "ZAWSZE robimy to wewnetrznie."
Pytanie: "Zawsze? Czy kiedykolwiek korzystaliscie z zewnetrznego wsparcia w innym obszarze?" (universal quantifier)
→ Otwiera drzwi do WYJATKU.
```

### 10 GOTOWYCH PYTAN META MODEL DLA CALLIOW

```
1. "Co DOKLADNIE ma Pan/Pani na mysli?" (specification — uniwersalne)
2. "Kto KONKRETNIE podjal te decyzje?" (referential index — identyfikuj decision makera)
3. "Co by sie stalo gdybyscie to ZROBILI?" (modal operator — otwiera mozliwosci)
4. "Zawsze? Nigdy? Na pewno?" (universal quantifier — challenge generalizacji)
5. "W porownaniu do CZEGO?" (comparative deletion — odkrywaj benchmark klienta)
6. "Jak DOKLADNIE to zmierzylistcie?" (specification — odkrywaj metodologie)
7. "Co musialoby byc PRAWDA zeby to mialo sens?" (presupposition — odkrywaj warunki)
8. "Skad wiecie ze to NIE zadziala?" (mind reading challenge — odkrywaj przekonania)
9. "Kiedy OSTATNIO to sprawdzaliscie?" (temporal — odkrywaj aktualnosc danych)
10. "Kto jeszcze w organizacji widzi to inaczej?" (referential — identyfikuj sojusznikow)
```
